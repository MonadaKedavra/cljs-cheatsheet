В ClojureScript все данные неизменяемы по умолчанию. Это значит, что значение символа не может быть изменено после его определения.

В операторах ветвления все значения приводятся к true за исключением false и nil.

В JavaScript правила приведения типов намного сложнее.

ИмяКодБулевое значение
Пустая строка""true
Ноль0true
Не числоjs/NaNtrue
Пустой вектор[]true
Пустой массив(array)true
Falsefalsefalse
Nilnilfalse

Все числа в ClojureScript представлены в формате числа двойной точности с плавающей запятой стандарта IEEE 754. Так же, как и в JavaScript.

Атомы предоставляют возможность управления состоянием в ClojureScript.

В отличие от JavaScript, в ClojureScript все данные неизменяемы по умолчанию. Это значит, что вы не можете изменить значение чего-то после определения.

Атомы делают возможным изменение значения. Также они разделяют то, как значение изменяется и считывается, что делает состояние более понятным.

Наблюдающие функции выполняются, когда значение изменяется.Это идеальное поведение, когда данные являются отображением UI.

Функции в ClojureScript и в JavaScript схожи.

Базовая библиотека имеет множество полезных функций высшего порядка. Также в ClojureScript есть удобное сокращение для создания анонимных функций.

Короткая запись анонимных функций #() очень удобна для создания небольших функций.

Анонимные функции #() не могут быть вложенными, и считается правильным писать их как можно короче.

СокращениеРаскрывается в
#(str "Hello " %)
(fn [n]
  (str "Hello " n))
#(my-fn %1 %2 %3)
(fn [a b c]
  (my-fn a b c))
#(* % (apply + %&))
(fn [x & the-rest]
  (* x (apply + the-rest)))

Строки в ClojureScript такие же, как в JavaScript, и у них есть все методы и свойства, которые есть у строк в JavaScript.

Строки в ClojureScript должны быть заключены в двойные кавычки.

Пространство имен clojure.string содержит много полезных функций для работы со строками.

В ClojureScript есть четыре типа коллекций: списки, векторы, множества и словари. Каждый из этих типов данных имеет свои сильные стороны, и все они активно используются в большинстве программ.

Все коллекции являются неизменяемыми и постоянными, что означает, что при изменении они сохраняют свою предыдущую версию. Создание "измененной" версии любой коллекции является эффективной операцией.

У всех коллекций есть представление в виде литерала:

КоллекцияЛитерал
Список()
Вектор[]
Множество#{}
Словарь{}

Списки — это последовательности значений, так же, как и векторы.

Большинство литералов списков представляют собой вызов функции.

(a b c) список из трех значений, и в то же время это означает "вызов функции a с двумя аргументами: b и c"

Векторы представляют собой наборы значений, которые индексированы последовательными целыми числами.

Хоть они и похожи, но массивы в JavaScript не то же самое, что векторы в ClojureScript. (.indexOf my-vec) не сработает на векторе.

Вектор может быть использован как функция, для получения его значений.

Множества — это коллекции уникальных значений, так же, как и в математике.

Множество может быть использован как функция, для получения его значений.

Словарь — это коллекция пар ключ/значение. Получение значения по ключу из коллекции работает очень быстро.

В JavaScript, объекты обычно используются как словари, в которых ключами могут быть только строки. В ClojureScript ключом в словаре может быть любое значение, хотя принято использовать специальный тип данных Keyword (ключевое слово).

Ключевые слова могут быть использованы как функции для получения значений из словаря. Поэтому они часто используются как ключи в словарях.

Многие базовые операции определены на последовательностях. Последовательность представляет собой интерфейс списка.

Любая последовательность является коллекцией, а любая коллекция может быть преобразована в последовательность с помощью фцнкции seq. На самом деле, это то, что происходит с коллекцией, когда она попадает в функцию последовательности.

Большинство функций последовательностей ленивы, что означает, что они обрабатывают их элементы по мере необходимости. Это дает возможность создавать бесконечные последовательности.

Вы можете заставить последовательность вычислить все свои элементы, используя функцию doall. Это удобно, когда вы хотите узнать результат выполнения функции с побочными эффектами на всех элементах последовательности.
